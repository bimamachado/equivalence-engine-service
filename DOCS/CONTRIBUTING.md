# Contributing

Obrigado por contribuir! Siga estas diretrizes para agilizar revisões.

- Abra issues antes de grandes features.
- Faça PRs pequenos e focados.
- Escreva testes para novas funcionalidades.
- Garanta que `pytest` passe localmente.
- Formate código com `black` e rode `ruff`/linters.
- Use mensagens de commit claras (tipo/escopo: breve descrição).

Processo de PR
1. Fork -> branch -> commit -> PR
2. Descreva mudanças no corpo do PR e inclua instruções de teste.
3. Peça revisão de 1-2 colegas.
4. Após aprovação, rebase se necessário e merge.
